Q&A 2 How do you download and organize a public GWAS dataset in R?

2.1 Explanation

To support reproducible and cross-platform GWAS workflows, it‚Äôs best to download genotype and phenotype data using R functions. We use a real rice diversity panel from Zhao et al.¬†2011, store all files with a rice_gwas_ prefix, and convert the phenotype file to both .csv and .tsv formats using tidyverse tools.

2.2 R Code

# üìö Load modern R packages
library(readr)
library(fs)
library(glue)

# üìÅ Ensure the data folder exists
dir_create("data")

# üì¶ Download and unzip genotype data
geno_zip_url <- "http://ricediversity.org/data/sets/44kgwas/RiceDiversity.44K.MSU6.Genotypes_PLINK.zip"
geno_zip_path <- "data/rice_gwas_genotypes.zip"

download.file(geno_zip_url, destfile = geno_zip_path, mode = "wb")
unzip(geno_zip_path, exdir = "data/temp_geno")

# üìÑ Move genotype files and rename with prefix
geno_files <- dir_ls("data/temp_geno", regexp = "\\.(ped|map|bed|bim|fam)$")
for (file in geno_files) {
  ext <- path_ext(file)
  file_move(file, glue("data/rice_gwas_genotypes.{ext}"))
}

# üßπ Clean up
dir_delete("data/temp_geno")
file_delete(geno_zip_path)

# üìë Download phenotype data
pheno_url <- "http://www.ricediversity.org/data/sets/44kgwas/RiceDiversity_44K_Phenotypes_34traits_PLINK.txt"
pheno_txt <- "data/rice_gwas_phenotypes.txt"
download.file(pheno_url, destfile = pheno_txt, mode = "wb")

# üìÇ Load and convert to CSV + TSV
phenotypes <- read_table(pheno_txt)  # auto-detects delimiter
write_csv(phenotypes, "data/rice_gwas_phenotypes.csv")
write_tsv(phenotypes, "data/rice_gwas_phenotypes.tsv")

# üßπ Optionally remove the raw .txt version
file_delete(pheno_txt)

message("‚úÖ Rice GWAS dataset downloaded and formatted for analysis.")

‚úÖ Takeaway: Using modern tools like fs, readr, and glue, you can automate GWAS data preparation in a clean and reproducible format ready for downstream analysis.
