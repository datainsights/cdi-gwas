Q&A 6 How do you assign sample and SNP identifiers and align genotype data with phenotype records?

6.1 Explanation

After converting genotype data to a matrix, the next step is to label rows and columns using the original .fam and .map files:

  • .fam provides sample identifiers (usually the second column = IID)
  • .map provides SNP identifiers (usually the second column = SNP ID)

We‚Äôll load those files, assign them as rownames() and colnames(), and then verify that the sample IDs match the phenotype data ‚Äî ensuring both data types are aligned for analysis.

6.2 R Code

library(tidyverse)
library(rrBLUP)
library(BGLR)
library(DT)
library(SNPRelate)
library(dplyr)
library(qqman)
library(poolr)

# üß¨ Load genotype matrix
Geno <- readRDS("RDataRDS/geno_matrix.rds")

# üìë Load sample and SNP info
FAM <- read_table("data/sativa413.fam", col_names = FALSE, show_col_types = FALSE)
MAP <- read_table("data/sativa413.map", col_names = FALSE, show_col_types = FALSE)

# üè∑Ô∏è Assign row and column names
rownames(Geno) <- FAM$X2        # Individual IDs
colnames(Geno) <- MAP$X2        # SNP IDs

# üìÑ Load phenotype data
phenotypes <- read_delim("data/sativa_phenotypes.tsv", show_col_types = FALSE)

# ‚úÖ Check alignment between genotype matrix and phenotype table
table(rownames(Geno) == phenotypes$NSFTVID)

‚úÖ Takeaway: Assigning identifiers from .fam and .map ensures interpretability of your genotype matrix. Verifying alignment with the phenotype table prevents mismatches and guarantees that downstream analyses are valid and reproducible.
